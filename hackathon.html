Hackathon

In collaboration with MLOps Community

Feb 7 - Mar 4, 2022

Come build MLOps and Data products, have some fun, and win exciting prizes!

Challenge

Join us for a 4-week virtual hackathon starting Feb 7!

ML applications are built by many, but production-ready ML applications are delivered by a few. Do you have the urge to venture into the MLOps space to get the hang of how ML-in-production works in real-time? Let this hackathon help you attain the required skill set to be a know-all ML geek. The theme is, as you may have guessed, building end-to-end ML applications with Flyte as the MLOps base.

Your idea could be around any ML or Data application. Pick up retail use cases, fraud detection, time-series forecasting problems, feature engineering, or literally, any ML or data use case you think adds value when made production-ready, and we are good to go!

This hackathon provides a chance to contribute to open source, interact with experienced MLOps professionals, and have fun building applications in the process.

How to Compete

The application you build has to be a part of the Flytelab repository. You should:

  • Fork the Flytelab repo and clone it
  • Create a new project on Flytelab by taking this template as a reference
  • Implement the application

Steps to implement the application from a workflow and app perspective are outlined below:

Workflow iteration cycle

  • Write code
  • Debug locally on sandbox
  • Create a PR (this automatically kicks off workflow registration)
  • Merge to master
  • Auto-deploy workflows to cluster

[Optional] App iteration cycle

  • Write code
  • Debug locally on local streamlit server
  • Merge to master
  • Streamlit cloud auto-deploy

The code will be reviewed on a regular basis, and the team will provide you the feedback.

After the acceptance of your application, you will:

  • Present the application at Flyte/MLOps meetup
  • Write a blog post
  • Give an MLOps community coffee session interview
